NPM og Local Modules

Hvad:
NPM står for Node Package Manager. Det er verdens største software register, og det indeholder over 800.000 software pakker. Det kaldes for lokale moduler, da det er pakker man henter og opbevarer som dependencies og pakker i sit projekt, i modsætningen til de globale, som man kan gøre brug af uden at hente pakker ned.

Hvorfor:
Man kan derfor benytte NPM til at hente software pakker, som man ønsker at gøre brug af i sin applikation. Det er gratis at benytte sig af, samt open-source. Grunden til at det er så populært, er at det er både er nemt, enkelt og utroligt hurtigt at gøre brug af NPM. NPM er også brugbart, da der er mange ting som de fleste applikationer gør brug af, dette kan f.eks. være validaton af email. Her kan man benytte sig af en af NPM software pakkerne, som kan gøre arbejdet for en.

Hvordan:
For at kunne bruge NPM, skal man have Node.js installeret på forhånd. Når man installerer node, får man også adgang til NPM. Så skal man initiere NPM til projektet, dette gør man med kommandoen ”npm init” i terminalen. Dette vil skabe en helt ny fil til projektet, som er en ”package.json” fil. package.json filen indeholder dependencies, og det er her de fremtidige npm pakker som man gør brug af, kommer til at ses. Når projektet er initieret og json filen er skabt, kan man installere de NPM pakker, som man ønsker at gøre brug af. Man bruger terminalen, hvor man kan skrive kommandoer ind som ønskes eksekveret. For at installere en NPM software pakke skriver man i terminalen: npm install efterfulgt af pakkens navn.

Node Core Modules

Module system: Require funktionen – loader moduler ind i applikationen. Require funktionen benyttes, til at loade modulerne ind i systemet, så man kan benytte dem i sin applikation. Hvis vi ønsker at benytte metoderne under f.eks. FilePrinter modulet, skal vi først benytte require funktionen til at hente modulet ind i applikationen. Require funktionen returnerer alt fra modulet som vi har angivet. Det skal gemmes i en variabel således: const fs = require(’fs’) Nu kan man bruge metoderne fra modulet FilePrinter (fs), og dets dertilhørende metoder. F.eks. er writeFileSync() og appendFileSync() en del af FilePrinter modulet. Nu kan vi bruge metoderne, ved at binde den til const fs, som indeholder require(’fs’). Her er et eksempel. const fs = require(’fs’) fs.writeFileSync(’eksempel.txt’, ’noget valgfri tekst som man vil have gemt i sin tekst fil’) Bemærk at metoden writeFineSync tager 2 parameter – den vil vide hvilken fil den skal skrive til, og hvad den skal skrive. Man kan læse mere om de forskellige moduler og hvilke metoder de indeholder, inde på hjemmesiden: www.nodejs.org

Funktioner

Functions er objekter der kan blive manipuleret med. Extended og sendt rundt som data. En function ville normalt se ud som:
function functionName() {
// function body
// optional return;
}
alle funtioner retunerer en værdi

Callbacks

En call er en function som sættes i en andens function parameter og bliver eksekvere i den pågældende function. Bruges ofes i asynchronous programmering. Man bruger callbacks når man f.eks. lavet et API call. Mange requests uden at skulle vente på en function retunerer en værdi.

Express

Express er et hurtigt framework som bruges af Node.js, det hentes ind i projektet ved hjælp af npm install.
